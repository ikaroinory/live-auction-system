### 2.2 关键设计

#### 2.2.1 数据库设计

**新增 Bid 表**：

| 字段名 | 类型 | 约束 | 说明 |
| :--- | :--- | :--- | :--- |
| id | VARCHAR(36) | PRIMARY KEY | 出价记录ID |
| productId | VARCHAR(36) | NOT NULL, FOREIGN KEY | 商品ID |
| userId | VARCHAR(36) | NOT NULL, FOREIGN KEY | 用户ID |
| price | DECIMAL(10,2) | NOT NULL | 出价金额 |
| createdAt | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**修改 Product 表**（已有）：
- `currentBidPrice` - 当前出价
- `bidCount` - 出价次数

#### 2.2.2 WebSocket消息协议

| 消息类型 | 方向 | 说明 |
| :--- | :--- | :--- |
| JOIN_ROOM | 客户端→服务端 | 加入竞拍房间 |
| LEAVE_ROOM | 客户端→服务端 | 离开竞拍房间 |
| SUBMIT_BID | 客户端→服务端 | 提交出价 |
| BID_SUCCESS | 服务端→客户端 | 出价成功 |
| BID_FAILED | 服务端→客户端 | 出价失败 |
| PRICE_UPDATE | 服务端→客户端 | 价格更新 |
| RANKING_UPDATE | 服务端→客户端 | 排行榜更新 |
| COUNTDOWN_TICK | 服务端→客户端 | 倒计时更新 |
| AUCTION_ENDED | 服务端→客户端 | 竞拍结束 |

#### 2.2.3 API接口设计

| API路径 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/v1/bids` | GET | 获取出价记录列表 |
| `/api/v1/bids/{id}` | GET | 获取出价记录详情 |
| `/api/v1/bids` | POST | 创建出价记录 |
| `/api/v1/products/{id}/bids` | GET | 获取商品出价记录 |

## 4. 代码安全性

### 4.1 注意事项

| 序号 | 风险点 | 风险等级 | 关联模块 |
| :--- | :--- | :--- | :--- |
| 1 | 恶意出价攻击（重复出价、低价出价） | 高 | 后端出价验证 |
| 2 | WebSocket连接伪造 | 高 | WebSocket认证 |
| 3 | 竞态条件导致数据不一致 | 高 | 出价处理 |
| 4 | 价格计算溢出 | 中 | 价格计算逻辑 |
| 5 | SQL注入 | 中 | 数据库操作 |
| 6 | 用户身份伪造 | 高 | 认证机制 |

### 4.2 解决方案

| 风险点 | 解决方案 | 实施位置 |
| :--- | :--- | :--- |
| 恶意出价攻击 | 使用Redis分布式锁防止重复出价，验证价格必须高于当前价+最小加价 | 后端服务 |
| WebSocket连接伪造 | 连接时验证用户token | WebSocket服务 |
| 竞态条件 | 使用Redis原子操作 + 消息队列异步写入 | 后端服务 |
| 价格计算溢出 | 使用Decimal类型，添加数值范围检查 | 后端服务 |
| SQL注入 | 使用Prisma ORM参数化查询 | 后端服务 |
| 用户身份伪造 | JWT token认证，验证用户存在 | 中间件 |
