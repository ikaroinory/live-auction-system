📊 项目概述

在直播电商生态中，珠宝、二手奢侈品等高价值非标品难以统一定价，而"动态竞拍"能通过强互动与竞技感最大化商品价值。本系统聚焦高并发与实时交互场景，基于 React、WebSocket 与 Node.js 等全栈技术，打造完整的直播竞拍闭环系统，实现从竞拍商品上架、自定义规则设定、实时出价参与到最终落锤成交的全链路流转。

🎯
核心功能
  • 实时直播竞拍：基于WebSocket实现低延迟推送，支持高并发实时竞拍场景设计
  • 商品讲解上架：主播实时讲解，一键上架竞拍
  • 分布式并发控制：Redis分布式锁防冲突
  • 延时竞拍机制：出价后延时指定秒数
  • 直播间互动：聊天、点赞特效

🚀 工程难点与解决方案

并发出价冲突

问题描述：高并发场景下，多个用户同时出价可能导致价格不一致、数据冲突，影响竞拍公平性

解决方案：Redis分布式锁

  • 使用 SET lockKey value NX PX 1000 确保锁的唯一性
  • NX参数：只在key不存在时设置，保证互斥
  • PX参数：设置1秒自动过期，防止死锁
  • try-finally机制确保锁一定会释放，保证高可用性

WebSocket稳定性

问题描述：网络波动导致连接断开、消息丢失，需要断线重连和心跳检测机制

解决方案：Socket.io高可用方案

  • 内置自动重连机制，指数退避策略避免风暴
  • 支持WebSocket和HTTP轮询双模式自动切换
  • 心跳检测每10秒检测连接状态，超时自动重连
  • 消息队列缓存离线消息，重连后自动补发

延时竞拍机制

问题描述：需要精确控制延时时间，确保所有用户看到相同倒计时，防止恶意竞拍

解决方案：定时器+Redis存储

  • 服务端定时器每秒推送COUNTDOWN_TICK事件
  • Redis存储竞拍结束时间戳，确保全局一致性
  • 出价后自动延长指定秒数，防止最后时刻恶意竞拍
  • 客户端根据服务器时间戳计算本地倒计时，避免时钟偏差
