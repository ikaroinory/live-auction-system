# 抖音电商AI全栈课题 - 直播竞拍全栈系统 项目结构

## 1. 项目概述

**项目名称**：实时竞拍大师 - 抖音电商直播竞拍全栈系统

**核心目标**：构建完整的直播电商实时竞拍系统，实现「商品上架 → 规则配置 → 实时出价 → 动态排名 → 竞拍成交」全流程闭环，支持高并发场景下的毫秒级实时同步。

---

## 2. 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React 18 + TypeScript | 组件化开发，类型安全 |
| 状态管理 | Zustand / Redux Toolkit | 全局状态统一管理 |
| 实时通信 | Socket.io / 原生 WebSocket | 长连接，支持心跳保活与自动重连 |
| UI 组件库 | Ant Design Mobile / 自定义组件 | 移动端 H5 适配 |
| 后端 | Go | 高性能异步处理 |
| 数据库 | MySQL | 持久化存储核心业务数据 |
| 缓存层 | Redis 6+ | 高频读写、实时状态、分布式锁 |
| 消息队列 | Kafka  | WebSocket 消息广播 |
| 部署 | Docker + Docker Compose | 容器化一键部署 |

## 6. 数据库设计

### 6.1 核心表结构

**auctions 竞拍商品表**
```sql
CREATE TABLE auctions (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  seller_id BIGINT NOT NULL,
  title VARCHAR(255) NOT NULL,
  description TEXT,
  images JSON,
  start_price DECIMAL(10,2) NOT NULL DEFAULT 0,
  min_increment DECIMAL(10,2) NOT NULL,
  max_price DECIMAL(10,2),
  duration_seconds INT NOT NULL,
  auto_extend_seconds INT DEFAULT 15,
  status TINYINT NOT NULL DEFAULT 0, -- 0:未开始 1:进行中 2:已结束 3:已取消
  start_time DATETIME,
  end_time DATETIME,
  final_price DECIMAL(10,2),
  winner_id BIGINT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  INDEX idx_status (status),
  INDEX idx_start_time (start_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**bids 出价记录表**
```sql
CREATE TABLE bids (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  auction_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  price DECIMAL(10,2) NOT NULL,
  created_at DATETIME(3) DEFAULT CURRENT_TIMESTAMP(3),
  UNIQUE KEY uk_auction_user (auction_id, user_id, price),
  INDEX idx_auction_time (auction_id, created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**orders 订单表**
```sql
CREATE TABLE orders (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  auction_id BIGINT NOT NULL,
  user_id BIGINT NOT NULL,
  seller_id BIGINT NOT NULL,
  final_price DECIMAL(10,2) NOT NULL,
  status TINYINT NOT NULL DEFAULT 0, -- 0:待支付 1:已支付 2:已取消
  paid_at DATETIME,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 7. Redis 数据结构设计

| Key 模式 | 类型 | 用途 |
|----------|------|------|
| `auction:running:{auctionId}` | Hash | 进行中竞拍的实时状态（当前价、结束时间等） |
| `auction:bids:{auctionId}` | Sorted Set | 出价排行榜，按价格排序 |
| `auction:lock:{auctionId}` | String | 分布式锁，防止并发出价冲突 |
| `ws:room:{auctionId}:users` | Set | 直播间在线用户集合 |
| `auction:user:bid:{userId}:{auctionId}` | String | 用户最新出价缓存 |

## 8. 核心竞拍规则实现逻辑

```typescript
// 竞拍规则引擎伪代码
class AuctionRuleEngine {
  // 0元起拍
  validateStartPrice(price: number): boolean {
    return price >= 0;
  }

  // 加价幅度校验
  validateIncrement(currentPrice: number, newPrice: number, minIncrement: number): boolean {
    return newPrice >= currentPrice + minIncrement;
  }

  // 封顶价校验
  validateMaxPrice(newPrice: number, maxPrice: number | null): boolean {
    if (!maxPrice) return true;
    return newPrice <= maxPrice;
  }

  // 自动延时逻辑
  calculateNewEndTime(originalEndTime: Date, extendSeconds: number): Date {
    const now = new Date();
    const remainingMs = originalEndTime.getTime() - now.getTime();
    // 结束前30秒内有人出价才触发延时
    if (remainingMs > 0 && remainingMs < 30 * 1000) {
      return new Date(originalEndTime.getTime() + extendSeconds * 1000);
    }
    return originalEndTime;
  }
}
```

## 9. WebSocket 消息协议定义

| 消息类型 | 方向 | 说明 |
|----------|------|------|
| `JOIN_ROOM` | C→S | 用户加入直播间 |
| `LEAVE_ROOM` | C→S | 用户离开直播间 |

## 10. 高并发优化方案

1. **Redis 分层缓存策略**：竞拍实时状态全量放 Redis，定时异步持久化到 MySQL
2. **分布式锁**：每个竞拍 ID 独立锁，保证出价幂等性，绝对不允许重复出价
3. **WebSocket 房间级路由隔离**：不同直播间的消息互不干扰，单直播间支持 1000+ 用户同时在线
4. **读写分离**：出价写入主库，历史查询走从库
5. **消息削峰**：高频出价通过 Redis Pub/Sub 异步广播，避免阻塞主线程
