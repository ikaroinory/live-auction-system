# 直播竞拍系统项目结构

## 一、系统概述

本系统是一个面向高价值非标品（珠宝、二手奢侈品等）的直播竞拍平台，支持毫秒级实时出价、动态排名计算、AI辅助定价等核心功能。

### 技术栈
- **前端**：React 18 + TypeScript + Zustand + TailwindCSS
- **后端**：Go 1.21+ + Gin + Gorilla WebSocket + GORM
- **实时通信**：WebSocket（双向通信）
- **数据库**：PostgreSQL（主库）+ Redis（缓存/消息队列）
- **AI服务**：Python FastAPI（独立微服务）

type Auction struct {
    ID            string        `json:"id"`

    StartPrice    float64       `json:"start_price"`

}

// bid.go - 出价实体
type Bid struct {
    ID        string    `json:"id"`
    AuctionID string    `json:"auction_id"`
    UserID    string    `json:"user_id"`
    Amount    float64   `json:"amount"`
    BidTime   time.Time `json:"bid_time"`

type IncrementRule struct {
    MinIncrement   float64 `json:"min_increment"`

## 六、数据库设计

### 核心表结构

```sql
-- 用户表
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) DEFAULT 'buyer', -- buyer, seller, streamer, admin
    avatar_url VARCHAR(255),
    balance DECIMAL(12,2) DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 商品表
CREATE TABLE products (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    seller_id UUID REFERENCES users(id),
    name VARCHAR(200) NOT NULL,
    description TEXT,
    category VARCHAR(50),
    images JSONB,
    certificate_no VARCHAR(100), -- 鉴定证书编号
    status VARCHAR(20) DEFAULT 'pending', -- pending, approved, rejected
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 竞拍表
CREATE TABLE auctions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    product_id UUID REFERENCES products(id),
    live_room_id UUID REFERENCES live_rooms(id),
    start_price DECIMAL(12,2) NOT NULL,
    current_price DECIMAL(12,2) DEFAULT 0,
    reserve_price DECIMAL(12,2),
    increment_rule JSONB,
    start_time TIMESTAMP NOT NULL,
    end_time TIMESTAMP NOT NULL,
    status VARCHAR(20) DEFAULT 'pending', -- pending, live, ended, sold, passed
    winner_id UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- 出价记录表
CREATE TABLE bids (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    auction_id UUID REFERENCES auctions(id),
    user_id UUID REFERENCES users(id),
    amount DECIMAL(12,2) NOT NULL,
    bid_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    is_winning BOOLEAN DEFAULT FALSE,
    INDEX idx_auction_bids (auction_id, bid_time DESC)
);

-- 直播间表
CREATE TABLE live_rooms (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    streamer_id UUID REFERENCES users(id),
    title VARCHAR(200),
    status VARCHAR(20) DEFAULT 'offline',
    viewer_count INTEGER DEFAULT 0,
    push_url VARCHAR(255),
    pull_url VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 七、WebSocket消息协议

### 消息格式

```typescript
interface WebSocketMessage {
  type: MessageType;
  action: string;
  payload: any;
  timestamp: number;
  seq: number; // 消息序号
}

enum MessageType {
  // 竞拍相关
  BID_PLACE = 'bid:place',           // 出价
  BID_ACCEPTED = 'bid:accepted',     // 出价被接受
  BID_REJECTED = 'bid:rejected',     // 出价被拒绝
  AUCTION_UPDATE = 'auction:update', // 竞拍状态更新
  AUCTION_END = 'auction:end',       // 竞拍结束
  
  // 直播相关
  CHAT_MESSAGE = 'chat:message',     // 聊天消息
  DANMAKU = 'danmaku',               // 弹幕
  VIEWER_COUNT = 'viewer:count',     // 观看人数
  
  // 系统消息
  HEARTBEAT = 'system:heartbeat',    // 心跳
  ERROR = 'system:error'             // 错误
}
```

## 十、技术难点与解决方案

| 难点 | 解决方案 |
|------|----------|
| 高并发出价 | Redis原子操作 + 消息队列削峰 |
| 状态一致性 | 分布式锁 + 乐观锁 + 事件溯源 |
| 毫秒级延迟 | WebSocket长连接 + 边缘节点部署 |
| 防狙击机制 | 延时落锤 + 随机延时 |
| 恶意出价检测 | AI风控模型 + 行为分析 |
