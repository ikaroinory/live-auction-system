## 核心功能清单

### 实时直播竞拍
- **功能描述**：通过WebSocket实现实时竞拍通信，支持多人同时在线竞拍
- **技术实现**：
  - Socket.io实现WebSocket服务端和客户端
  - 竞拍房间管理（JOIN_ROOM、LEAVE_ROOM）
  - 实时价格推送（PRICE_UPDATE）
  - 实时排名更新（RANKING_UPDATE）
  - 实时倒计时推送（COUNTDOWN_TICK）
  - 竞拍结束通知（AUCTION_ENDED）
- **用户体验**：类似抖音直播竞拍的真实体验，价格、排名、倒计时实时更新，无延迟

### 商品讲解与上架
- **功能描述**：主播可实时讲解商品，一键上架开始竞拍
- **技术实现**：
  - 商品创建与编辑（Admin后台）
  - 商品状态管理（PENDING待上架、PUBLISHED已上架）
  - 商品讲解状态管理（Redis存储当前讲解商品ID）
  - WebSocket推送讲解状态更新（EXPLAINING_UPDATE）
  - 商品竞拍状态管理（NOT_STARTED未开始、IN_PROGRESS进行中、ENDED已结束）
- **用户体验**：主播可在直播过程中实时讲解商品，用户H5端实时看到讲解商品信息

### 分布式并发控制
- **功能描述**：Redis分布式锁防止并发出价冲突，保证竞拍公平性
- **技术实现**：
  - Redis分布式锁（`auction:lock:${productId}`）
  - 出价前获取锁，锁超时时间1秒
  - 出价完成后释放锁
  - 防止同一商品同一时刻多个出价操作并发执行
- **技术优势**：保证竞拍公平性，防止价格不一致或数据冲突

### 延时竞拍机制
- **功能描述**：出价后自动延时，防止最后时刻恶意竞拍
- **技术实现**：
  - 每次出价成功后，竞拍结束时间延长`extendSeconds`秒（默认15秒）
  - 定时器每秒推送倒计时（COUNTDOWN_TICK）
  - 竞拍结束时自动确定获胜者（最高价用户）
- **用户体验**：防止恶意竞拍，提升竞拍公平性和用户参与度

### 直播间互动
- **功能描述**：实时聊天、点赞、礼物动画等互动功能
- **技术实现**：
  - 实时聊天消息展示（模拟聊天消息）
  - 点赞动画效果（❤️图标上浮动画）
  - 用户关注直播间功能
- **用户体验**：增强直播间互动氛围，提升用户参与感

### 用户认证与管理
- **功能描述**：支持手机号注册登录、短信验证码登录、用户信息管理
- **技术实现**：
  - 手机号+密码注册登录
  - 短信验证码登录（测试环境验证码：123456或666666）
  - JWT token生成与验证
  - 用户信息管理（头像、昵称、个人简介、性别、生日、所在地）
  - IP地址解析用户所在地
- **用户体验**：多种登录方式，便捷的用户信息管理

## 端到端使用流程

1. **用户进入首页**：用户通过H5端进入首页，浏览正在直播的直播间列表，可上下滑动切换直播间

2. **进入直播间**：用户点击直播间卡片进入直播间页面，实时观看直播内容（视频播放区）并参与互动（聊天、点赞）

3. **主播准备商品**：主播在Admin后台登录后，进入商品管理页面，创建新商品并填写商品信息（名称、图片、起拍价、加价幅度、竞拍时长等）

4. **商品上架与讲解**：主播将商品状态改为"已上架"，点击"开始讲解"按钮，商品在H5端直播间实时展示，主播讲解商品详情

5. **开始竞拍**：主播点击"开始竞拍"按钮，商品进入竞拍状态，倒计时开始（默认60分钟），WebSocket实时推送竞拍开始通知

6. **用户参与竞拍**：用户在H5端实时查看商品信息和当前价格，点击出价按钮输入出价金额，系统验证出价有效性（必须高于当前价格+固定加价）

7. **实时价格更新**：用户出价成功后，系统通过WebSocket实时推送价格更新、排名变化和倒计时信息，所有在线用户实时看到最新价格和排名

8. **延时机制**：每次出价成功后，竞拍结束时间自动延长15秒，防止最后时刻恶意竞拍

9. **竞拍结束**：倒计时结束后，系统自动确定获胜者（最高价用户），推送竞拍结束通知，更新商品状态为"已结束"

10. **查看竞拍记录**：用户可在个人中心查看竞拍记录和订单信息，主播可在Admin后台查看所有出价记录

## README / 运行说明

### 项目简介

**抖音电商直播竞拍全栈系统**是一个完整的直播竞拍平台，模拟抖音电商直播场景，支持主播实时讲解商品、用户在线竞拍、实时价格更新等核心功能。系统采用前后端分离架构，包含三个主要部分：

- **后端服务**：Express + TypeScript + Prisma + MySQL + Redis + Socket.io
- **H5前端**：React + TypeScript + Antd Mobile + Socket.io-client（移动端用户界面）
- **Admin前端**：React + TypeScript + Semi Design + Socket.io-client（主播管理后台）

## 大模型 / AI 能力使用说明

**本项目未使用大模型或AI能力。**

### 说明

本系统为传统全栈开发项目，采用成熟的技术栈（Express、React、MySQL、Redis、WebSocket）实现直播竞拍功能，未集成任何大模型、AI模型、Agent、RAG、向量库或Prompt方案。

### 技术选择原因

1. **业务场景明确**：直播竞拍是一个明确的业务场景，不需要AI辅助决策或内容生成
2. **实时性要求高**：竞拍场景对实时性要求极高，传统WebSocket方案更稳定可靠
3. **并发控制复杂**：竞拍场景需要精确的并发控制，Redis分布式锁方案成熟可靠
4. **开发成本可控**：传统技术栈开发成本更低，维护更简单

### 未来AI扩展可能性

如需扩展AI能力，可考虑以下方向：
1. **智能商品推荐**：基于用户历史竞拍记录，推荐相关商品
2. **竞拍价格预测**：基于历史数据，预测商品最终成交价格
3. **智能客服**：AI客服回答用户竞拍相关问题
4. **内容审核**：AI审核聊天内容，防止违规信息

## 关键工程难点与解决方案

### 难点1：并发出价冲突问题

#### 难点描述
在直播竞拍场景中，多个用户可能同时出价，如果并发控制不当，会导致：
- 价格不一致：多个出价同时写入数据库，导致当前价格混乱
- 数据冲突：同一时刻多个出价记录，无法确定有效出价
- 竞拍不公平：后出价的用户可能覆盖前出价用户的价格

#### 解决方案
采用Redis分布式锁机制，确保同一商品同一时刻只有一个出价操作能执行：

**技术优势**：
- Redis SET命令的NX参数确保锁的唯一性
- PX参数设置锁超时时间（1秒），防止死锁
- try-finally确保锁一定会释放
- 分布式锁保证竞拍公平性，防止并发冲突

### 难点2：WebSocket实时通信稳定性

#### 难点描述
WebSocket实时通信存在以下稳定性问题：
- 连接不稳定：网络波动导致WebSocket连接断开
- 断线重连：用户断线后需要自动重连，恢复竞拍状态
- 消息丢失：网络中断期间可能丢失重要消息（价格更新、竞拍结束等）
- 心跳检测：需要定期检测连接状态，防止僵尸连接

#### 解决方案
采用Socket.io的自动重连机制和心跳检测：

**技术优势**：
- Socket.io自动重连机制，无需手动处理断线重连
- 支持WebSocket和HTTP轮询两种传输方式，兼容性更好
- 心跳检测定期检测连接状态，及时发现僵尸连接
- 重连成功后自动恢复竞拍状态，用户体验无缝衔接

### 难点3：竞拍延时机制实现

#### 难点描述
竞拍延时机制需要解决以下问题：
- 精确时间控制：每次出价后需要精确延长竞拍结束时间
- 倒计时推送：每秒推送倒计时，确保所有用户看到相同时间
- 延时公平性：延时时间需要合理，防止恶意延时或延时过短
- 状态同步：延时后需要同步所有用户的倒计时状态

#### 解决方案
采用定时器+Redis存储竞拍结束时间：

**技术优势**：
- 定时器每秒推送倒计时，确保时间精确控制
- 出价后自动延时，防止最后时刻恶意竞拍
- 延时时间可配置（默认15秒），适应不同竞拍场景
- 竞拍结束自动确定获胜者，逻辑清晰可靠

## 项目亮点 / 创新点

### 亮点1：实时竞拍体验

**亮点描述**：
通过WebSocket实时推送价格、排名、倒计时，用户体验流畅，类似抖音直播竞拍的真实体验。

**技术实现**：
- Socket.io实现WebSocket实时通信
- 实时价格推送（PRICE_UPDATE），所有用户同步看到最新价格
- 实时排名更新（RANKING_UPDATE），展示当前出价排名
- 实时倒计时推送（COUNTDOWN_TICK），每秒推送剩余时间
- 竞拍结束通知（AUCTION_ENDED），自动推送获胜者信息

**用户体验**：
- 无延迟：价格、排名、倒计时实时更新，无网络延迟
- 流畅交互：出价按钮、价格输入、动画效果流畅自然
- 真实感：类似抖音直播竞拍的真实体验，沉浸感强

### 亮点2：分布式并发控制

**亮点描述**：
Redis分布式锁保证竞拍公平性，防止并发冲突，技术方案成熟可靠。

**技术实现**：
- Redis SET命令实现分布式锁（NX参数确保唯一性）
- 锁超时时间1秒，防止死锁
- try-finally确保锁一定会释放
- 出价前获取锁，出价后释放锁，保证同一商品同一时刻只有一个出价操作

**技术优势**：
- 竞拍公平性：防止并发冲突，保证竞拍公平
- 数据一致性：防止价格不一致或数据冲突
- 成熟可靠：Redis分布式锁是成熟的并发控制方案，广泛应用于电商、金融等领域

### 亮点3：延时竞拍机制

**亮点描述**：
出价后自动延时，防止恶意竞拍，提升竞拍公平性和用户参与度。

**技术实现**：
- 每次出价成功后，竞拍结束时间延长15秒（可配置）
- 定时器每秒推送倒计时，确保时间精确控制
- 延时机制防止最后时刻恶意竞拍（如最后1秒连续出价）

**用户体验**：
- 公平竞拍：防止恶意竞拍，提升竞拍公平性
- 参与度提升：延时机制给更多用户参与机会，提升用户参与度
- 紧张感：倒计时延时增加竞拍紧张感和刺激感
